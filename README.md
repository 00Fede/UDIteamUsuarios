# UDIteamUsuarios

Historia de Usuario
Yo, como dueño del producto, necesito contar con un administrador web (CRUD)  de los usuarios del sistema, en donde voy a tener un listado de usuarios con sus nombre, apellidos, email, teléfono, usuario, contraseña y un estado de activo/inactivo. Necesito tener la posibilidad de filtrarlos por el usuario y correo. Debo tener la posibilidad de crear un nuevo usuario, de editar todos sus datos,  así como la posibilidad de eliminarlos lógicamente del sistema.

Criterios de aceptaciòn

Propia base de datos
La aplicacion debe tener un diseño acorde (imagenes y colores) con la compañìa
La aplicación debe insertar un usuario
La aplicacion debe actualizar algun dato existente de un usuario
La aplicacion debe eliminar logicamente un usuario
Cada usuario debe ser unico, solo un usuario por id
Consultar informaciòn del usuario por el nombre
El campo de contraseña no debe ser visible
